## 一、概述

### 1.1 文档目的

本文档描述如何在本地环境中部署和运行 AxVisor QEMU 自动测试系统，实现：
- 在 QEMU 虚拟化环境中测试 AxVisor hypervisor
- 支持多种 Guest 系统（ArceOS、Linux、NimbOS）
- 支持多种架构（aarch64、x86_64）

### 1.2 测试架构

**测试流程：**

1. 构建 AxVisor
2. 下载 Guest 镜像
3. 启动 QEMU，加载 AxVisor + Guest
4. 捕获串口输出
5. 通过 success_regex / fail_regex 判断测试结果

### 1.3 支持的测试组合

| 架构 | Guest 系统 | 状态 | 成功标志 |
|------|-----------|------|----------|
| aarch64 | ArceOS | ✅ 已验证 | `Hello, world!` |
| aarch64 | Linux | ✅ 已验证 | `test pass!` |
| x86_64 | NimbOS | ✅ 已验证（需 VT-x） | `usertests passed!` |

### 1.4 QEMU 与硬件测试对照

QEMU 环境在虚拟化层面复现硬件平台（ROC-RK3568-PC、飞腾派等）的自动测试能力，对比如下：

| 能力项 | 硬件测试（uboot） | QEMU 测试 | 说明 |
|--------|-------------------|-----------|------|
| **固件加载** | U-Boot loady / TFTP | QEMU 直接加载到内存 | ✅ 等效复现 |
| **AxVisor 启动** | 加载到 DDR 运行 | QEMU 加载 ELF 运行 | ✅ 等效复现 |
| **ArceOS Guest** | ✅ ROC-RK3568-PC、飞腾派 | ✅ aarch64-qemu | ✅ 已复现 |
| **Linux Guest** | ✅ ROC-RK3568-PC、飞腾派 | ✅ aarch64-qemu | ✅ 已复现 |
| **NimbOS Guest** | - | ✅ x86_64-qemu（需 Intel VT-x） | 硬件 x86 待 PXE |
| **串口日志捕获** | USB 转 TTL → 串口 | QEMU -nographic → stdout | ✅ 等效复现 |
| **成功/失败判断** | success_regex / fail_regex | 同 ostool 正则 | ✅ 一致 |
| **电源控制** | 继电器控制上电/断电 | ❌ 不适用 | QEMU 进程即生命周期 |
| **多板并行** | 多 Runner 多板 | CI matrix 多实例 | ✅ 可并行 |
| **PXE 网络启动** | x86 待实现 | ❌ 未实现 | 直接加载替代 |

**结论**：QEMU 环境已复现硬件测试的核心能力（固件加载、Guest 运行、日志捕获、结果判断），无需物理设备即可完成大部分验证。电源控制、真实硬件外设等无法在 QEMU 中复现。

## 二、环境要求

### 2.1 必需工具

| 工具 | 最低版本 | 安装方式 | 验证命令 |
|------|----------|----------|----------|
| QEMU (aarch64) | 7.0+ | `sudo apt-get install qemu-system-arm` | `qemu-system-aarch64 --version` |
| QEMU (x86_64) | 7.0+ | `sudo apt-get install qemu-system-x86` | `qemu-system-x86_64 --version` |
| Rust | 1.70+ | [rustup](https://rustup.rs/) | `rustc --version` |
| ostool | ^0.8 | `cargo +stable install ostool --version ^0.8` | `ostool --version` |
| pkg-config | - | `sudo apt-get install pkg-config` | `pkg-config --version` |
| libssl-dev | - | `sudo apt-get install libssl-dev` | - |

### 2.2 可选工具

| 工具 | 用途 | 安装方式 |
|------|------|----------|
| gcc-aarch64-linux-gnu | 从源码构建 aarch64 Guest | `sudo apt-get install gcc-aarch64-linux-gnu` |
| cargo-binutils | Rust 二进制工具 | `cargo install cargo-binutils` |
| libguestfs-tools | rootfs 镜像操作 | `sudo apt-get install libguestfs-tools` |

### 2.3 x86_64 特殊要求

**AxVisor x86_64 仅支持 Intel VT-x (VMX)，不支持 AMD-V (SVM)：**

| CPU 类型 | 虚拟化标志 | AxVisor x86_64 |
|----------|------------|----------------|
| Intel | vmx (VT-x) | ✅ 支持 |
| AMD | svm (AMD-V) | ❌ 不支持 |

**环境支持情况：**

| 环境 | 是否支持 | 说明 |
|------|----------|------|
| 物理 Linux 机（Intel + VT-x） | ✅ | 推荐验证环境 |
| WSL2（Intel 宿主机 + 嵌套虚拟化） | ✅ | 需配置 .wslconfig |
| WSL2（AMD 宿主机） | ❌ | 仅 svm，AxVisor 不支持 |
| WSL2（未启用嵌套虚拟化） | ❌ | 无 VMX 暴露给 Guest |
| 无 VT-x 的物理机 | ❌ | 无法运行 |

**检查 CPU 虚拟化支持：**

```bash
# 检查 CPU 类型：vmx=Intel，svm=AMD
grep flags /proc/cpuinfo | head -1 | tr ' ' '\n' | grep -E '^vmx$|^svm$'

# 若有 vmx：Intel CPU，可运行 NimbOS
# 若有 svm：AMD CPU，NimbOS 无法运行

# 检查 KVM 模块
lsmod | grep kvm

# 检查 /dev/kvm
ls -la /dev/kvm
```

**WSL2 启用嵌套虚拟化**（Intel 宿主机）：

在 Windows 用户目录创建 `%USERPROFILE%\.wslconfig`：

```ini
[wsl2]
nestedVirtualization=true
```

保存后执行 `wsl --shutdown`，重新打开 WSL2。

### 2.4 网络与磁盘

- **网络**：首次运行需从 GitHub 下载 Guest 镜像（2.5MB - 17MB）
- **磁盘**：建议预留至少 2GB 可用空间

**预期耗时**（首次运行）：构建 AxVisor 约 2–5 分钟，下载镜像约 1–3 分钟，单次测试约 30 秒–2 分钟。后续运行（镜像已缓存）单次测试约 30 秒–2 分钟。

### 2.5 Guest 镜像来源与版本

**镜像仓库**：[axvisor-guest](https://github.com/arceos-hypervisor/axvisor-guest)

**下载地址**：`https://github.com/arceos-hypervisor/axvisor-guest/releases/download/v0.0.20/<镜像名>.tar.gz`

**当前版本**：v0.0.20（由 xtask 中 `IMAGE_URL_BASE` 固定）

**QEMU 测试用镜像**：

| 镜像名 | 架构 | Guest 系统 | 大小约 | 解压后内容 |
|--------|------|-----------|--------|------------|
| qemu_aarch64_arceos | aarch64 | ArceOS | ~2.5MB | qemu-aarch64, rootfs.img |
| qemu_aarch64_linux | aarch64 | Linux | ~17MB | qemu-aarch64, rootfs.img |
| qemu_x86_64_nimbos | x86_64 | NimbOS | ~2MB | qemu-x86_64, qemu-x86_64_usertests, rootfs.img |

**下载方式**：

```bash
# 自动下载并解压到 /tmp/axvisor/<镜像名>/
cargo xtask image download <镜像名>

# 可选：指定输出目录
cargo xtask image download qemu_aarch64_arceos --output-dir ./images

# 列出可用镜像
cargo xtask image ls
```

**校验**：下载后使用 SHA-256 校验，校验失败会报错。

**更新策略**：镜像版本由 AxVisor 仓库的 xtask 代码决定。若 axvisor-guest 发布新版本，需在 axvisor 的 `xtask/src/image.rs` 中更新 `IMAGE_URL_BASE` 和对应 SHA256。

## 三、快速开始

### 3.1 获取源码

```bash
git clone https://github.com/arceos-hypervisor/axvisor.git
cd axvisor
```

### 3.2 安装 ostool

```bash
cargo +stable install ostool --version ^0.8
```

### 3.3 统一 setup 脚本

项目提供统一入口 `setup_qemu.sh`，支持 `--guest` 参数：

```bash
# 方式 1：直接指定 guest
./scripts/setup_qemu.sh arceos
./scripts/setup_qemu.sh linux
./scripts/setup_qemu.sh nimbos

# 方式 2：使用 --guest 参数
./scripts/setup_qemu.sh --guest arceos

# 帮助
./scripts/setup_qemu.sh --help
```

原有脚本仍可用：`setup_qemu_arceos.sh`、`setup_qemu_linux.sh`、`setup_qemu_nimbos.sh`（内部调用统一脚本）。

### 3.4 aarch64 ArceOS 测试

```bash
# 准备环境
./scripts/setup_qemu.sh arceos

# 运行测试
cargo xtask qemu \
  --build-config configs/board/qemu-aarch64.toml \
  --qemu-config .github/workflows/qemu-aarch64.toml \
  --vmconfigs configs/vms/arceos-aarch64-qemu-smp1.toml
```

**成功标志**：输出 `Hello, world!` 并提示 `Detected success pattern`

### 3.5 aarch64 Linux 测试

```bash
# 准备环境
./scripts/setup_qemu.sh linux

# 运行测试
cargo xtask qemu \
  --build-config configs/board/qemu-aarch64.toml \
  --qemu-config .github/workflows/qemu-aarch64.toml \
  --vmconfigs configs/vms/linux-aarch64-qemu-smp1.toml
```

**成功标志**：输出 `test pass!` 并提示 `Detected success pattern`

### 3.6 x86_64 NimbOS 测试（需 Intel VT-x）

```bash
# 准备环境
./scripts/setup_qemu.sh nimbos

# 运行测试（使用 KVM 配置，需 Intel CPU + VT-x）
cargo xtask qemu \
  --build-config configs/board/qemu-x86_64.toml \
  --qemu-config .github/workflows/qemu-x86_64-kvm.toml \
  --vmconfigs configs/vms/nimbos-x86_64-qemu-smp1.toml
```

**成功标志**：输出 `usertests passed!` 并提示 `Detected success pattern`

**注意**：AMD CPU（仅 svm）无法运行，需 Intel CPU（vmx）。

## 四、详细部署步骤

### 4.1 aarch64 ArceOS 手动部署

#### 4.1.1 下载镜像

```bash
cargo xtask image download qemu_aarch64_arceos
ls -la /tmp/axvisor/qemu_aarch64_arceos
# 应包含：qemu-aarch64, rootfs.img
```

#### 4.1.2 修改 VM 配置

编辑 `configs/vms/arceos-aarch64-qemu-smp1.toml`：

```toml
[kernel]
image_location = "memory"
kernel_path = "/tmp/axvisor/qemu_aarch64_arceos/qemu-aarch64"
kernel_load_addr = 0x8020_0000
entry_point = 0x8020_0000
```

#### 4.1.3 准备 rootfs

```bash
mkdir -p tmp
cp /tmp/axvisor/qemu_aarch64_arceos/rootfs.img tmp/rootfs.img
```

#### 4.1.4 运行测试

```bash
cargo xtask qemu \
  --build-config configs/board/qemu-aarch64.toml \
  --qemu-config .github/workflows/qemu-aarch64.toml \
  --vmconfigs configs/vms/arceos-aarch64-qemu-smp1.toml
```

### 4.2 aarch64 Linux 手动部署

#### 4.2.1 下载镜像

```bash
cargo xtask image download qemu_aarch64_linux
ls -la /tmp/axvisor/qemu_aarch64_linux
```

#### 4.2.2 修改 VM 配置

编辑 `configs/vms/linux-aarch64-qemu-smp1.toml`：

```toml
[kernel]
image_location = "memory"
kernel_path = "/tmp/axvisor/qemu_aarch64_linux/qemu-aarch64"
kernel_load_addr = 0x8020_0000
entry_point = 0x8020_0000
```

#### 4.2.3 准备 rootfs

```bash
mkdir -p tmp
cp /tmp/axvisor/qemu_aarch64_linux/rootfs.img tmp/rootfs.img
```

#### 4.2.4 运行测试

```bash
cargo xtask qemu \
  --build-config configs/board/qemu-aarch64.toml \
  --qemu-config .github/workflows/qemu-aarch64.toml \
  --vmconfigs configs/vms/linux-aarch64-qemu-smp1.toml
```

### 4.3 x86_64 NimbOS 手动部署

#### 4.3.1 下载镜像

```bash
cargo xtask image download qemu_x86_64_nimbos
ls -la /tmp/axvisor/qemu_x86_64_nimbos
# 应包含：qemu-x86_64, qemu-x86_64_usertests, rootfs.img
```

#### 4.3.2 准备 rootfs（注入 usertests）

NimbOS 使用 `image_location = "fs"`，需要将内核注入到 rootfs 内部：

```bash
# 需要 libguestfs-tools 或 sudo 权限
./scripts/setup_qemu.sh nimbos
```

#### 4.3.3 QEMU 配置选择

**KVM 模式**（需要 VT-x）：

```toml
# .github/workflows/qemu-x86_64-kvm.toml
args = [
  "-m", "128M",
  "-smp", "1",
  "-machine", "q35",
  "-device", "virtio-blk-pci,drive=disk0",
  "-drive", "id=disk0,if=none,format=raw,file=${workspaceFolder}/tmp/rootfs.img",
  "-nographic",
  "-accel", "kvm",
  "-cpu", "host",
]
```

**TCG 模式**（无 VT-x，仅用于构建验证，运行会失败）：

```toml
# .github/workflows/qemu-x86_64.toml
args = [
  ...
  "-accel", "tcg",
  "-cpu", "qemu64",
]
```

#### 4.3.4 运行测试

```bash
cargo xtask qemu \
  --build-config configs/board/qemu-x86_64.toml \
  --qemu-config .github/workflows/qemu-x86_64-kvm.toml \
  --vmconfigs configs/vms/nimbos-x86_64-qemu-smp1.toml
```

### 4.4 切换 Guest 注意事项

ArceOS 和 Linux 共用 `tmp/rootfs.img`，**切换 Guest 前需重新运行对应的 setup 脚本**：

```bash
# 从 Linux 切回 ArceOS
./scripts/setup_qemu.sh arceos

# 从 ArceOS 切到 Linux
./scripts/setup_qemu.sh linux
```

## 五、配置文件说明

### 5.1 配置层次

| 层次 | 目录 | 用途 |
|------|------|------|
| 板级配置 | `configs/board/*.toml` | 构建配置（target、features） |
| QEMU 配置 | `.github/workflows/qemu-*.toml` | QEMU 启动参数、正则匹配 |
| VM 配置 | `configs/vms/*.toml` | Guest 定义（内核、内存、设备） |

### 5.2 板级配置示例

`configs/board/qemu-aarch64.toml`：

```toml
[build]
target = "aarch64-unknown-none"
features = ["axvisor/qemu"]
```

### 5.3 QEMU 配置示例

`.github/workflows/qemu-aarch64.toml`：

```toml
args = [
  "-m", "8g",
  "-smp", "4",
  "-machine", "virt,virtualization=on,gic-version=3",
  "-cpu", "cortex-a72",
  "-device", "virtio-blk-device,drive=disk0",
  "-drive", "id=disk0,if=none,format=raw,file=${workspaceFolder}/tmp/rootfs.img",
  "-nographic",
]
success_regex = ["Hello, world!", "test pass!"]
fail_regex = []
```

### 5.4 success_regex / fail_regex 说明

ostool 通过正则匹配 QEMU 串口输出来判断测试成功或失败，匹配到任一 success_regex 即成功，匹配到任一 fail_regex 即失败。

**当前配置**：

| 配置文件 | success_regex | fail_regex |
|----------|---------------|------------|
| qemu-aarch64.toml | `"Hello, world!"`, `"test pass!"` | `[]` |
| qemu-x86_64.toml | `"usertests passed!"` | `"System will reboot, press any key to continue ..."` |
| qemu-x86_64-kvm.toml | `"usertests passed!"` | `"System will reboot, press any key to continue ..."` |

**硬件测试（uboot.toml）参考**：

| 类型 | 正则 |
|------|------|
| success_regex | `"Welcome to AxVisor Shell!"`, `"All tests passed!"`, `"Hello World!"`, `"root@firefly:~#"`, `"root@phytium-Ubuntu:~#"`, `"Welcome to Phytium Buildroot"`, `"Last login: *"` |
| fail_regex | `"panicked at"` |

**扩展建议**：

1. **qemu-aarch64.toml 增加 fail_regex**：添加 `"panicked at"`，与 uboot 一致，便于及早发现崩溃。
2. **可选 success_regex**：若 Guest 输出 `"Welcome to AxVisor Shell!"`、`"All tests passed!"` 等，可加入以增强覆盖。
3. **正则格式**：支持 [regex](https://docs.rs/regex/latest/regex/) 语法，`*` 等需转义时注意。

### 5.5 VM 配置示例

`configs/vms/arceos-aarch64-qemu-smp1.toml`（节选）：

```toml
[base]
name = "arceos-qemu"
...

[kernel]
entry_point = 0x8020_0000
image_location = "memory"
kernel_path = "/tmp/axvisor/qemu_aarch64_arceos/qemu-aarch64"
kernel_load_addr = 0x8020_0000

[memory]
...

[devices]
...
```

## 六、CI 配置说明

### 6.1 触发条件

| 触发方式 | 说明 |
|----------|------|
| `push` | 推送到任意分支时触发 |
| `pull_request` | 创建或更新 PR 时触发 |
| `workflow_dispatch` | 在 GitHub Actions 页面手动触发 |

### 6.2 测试矩阵

`test-qemu.yml` 使用 matrix 策略并行运行 3 个测试用例：

| 架构 | Guest | VM 配置 | 镜像名 |
|------|-------|---------|--------|
| aarch64 | ArceOS | arceos-aarch64-qemu-smp1.toml | qemu_aarch64_arceos |
| aarch64 | Linux | linux-aarch64-qemu-smp1.toml | qemu_aarch64_linux |
| x86_64 | NimbOS | nimbos-x86_64-qemu-smp1.toml | qemu_x86_64_nimbos |

**失败处理**：`fail-fast: false`，单个用例失败不会取消其他用例。

### 6.3 Runner 要求

| 项目 | 要求 |
|------|------|
| **标签** | `[self-hosted, linux, intel]` |
| **aarch64** | 可运行 QEMU（TCG 或 KVM） |
| **x86_64** | 必须 Intel CPU + VT-x + KVM，AMD 无法通过 |

**注意**：仓库中 `qemu-x86_64.toml` 默认使用 TCG。若 Runner 支持 KVM，需在 CI 中改用 `qemu-x86_64-kvm.toml`，否则 NimbOS 测试会因无 VMX 而失败。

### 6.4 执行步骤

1. **Checkout**：检出仓库代码
2. **Install dependencies**：`cargo install ostool --version ^0.8`
3. **Download images and patch vm configs**：
   - 下载对应 Guest 镜像到 `/tmp/axvisor/<镜像名>/`
   - 根据 `image_location` 修补 VM 配置的 `kernel_path`
   - 若有 rootfs.img，修补 QEMU 配置中的 rootfs 路径；否则移除 rootfs 设备配置
4. **Run tests**：执行 `cargo xtask qemu`

### 6.5 镜像与配置修补逻辑

| image_location | 修补动作 |
|----------------|----------|
| `memory` | 更新 VM 配置 `kernel_path` 为 `/tmp/axvisor/<镜像名>/qemu-<arch>` |
| `fs` | 不修改 kernel_path。**NimbOS 使用 fs**：需将内核注入 rootfs 内部，CI 当前**未**自动执行 rootfs 注入，需在 Runner 上预先运行 `setup_qemu.sh nimbos` 或手动注入；否则 NimbOS 测试会因找不到内核而失败 |

rootfs 处理：若存在 `rootfs.img`，将 QEMU 配置中的 `file=${workspaceFolder}/tmp/rootfs.img` 替换为实际路径；否则移除 rootfs 相关参数。

## 七、故障排查

### 7.1 镜像下载失败

**现象**：`cargo xtask image download` 报错

**解决方案**：
1. 检查网络：`ping github.com`
2. 配置代理：
   ```bash
   export http_proxy=http://proxy:8080
   export https_proxy=http://proxy:8080
   ```
3. 手动下载：从 [axvisor-guest releases](https://github.com/arceos-hypervisor/axvisor-guest/releases) 下载并解压到 `/tmp/axvisor/<镜像名>/`

### 7.2 找不到 kernel_path 或 rootfs

**现象**：`ERROR: kernel image not found` 或 `Could not open 'tmp/rootfs.img'`

**解决方案**：
1. 确认已运行对应的 setup 脚本
2. 检查 VM 配置中 `kernel_path` 是否正确
3. 确认 `tmp/rootfs.img` 存在

### 7.3 ostool 版本不匹配

**现象**：`cargo xtask qemu` 报错

**解决方案**：
```bash
cargo +stable install ostool --version ^0.8
cargo install --list | grep ostool
```

### 7.4 未检测到成功模式

**现象**：QEMU 超时退出，未出现 `Detected success pattern`

**解决方案**：
1. 查看完整 QEMU 输出，确认 Guest 是否正常启动
2. 检查 `success_regex` 配置是否与实际输出匹配
3. 增加 QEMU 内存（`-m` 参数）

### 7.5 x86_64 VMX 不支持

**现象**：
```
Hardware support: false
CPU does not support feature VMX
Failed to enable virtualization: Unsupported
```

**可能原因与解决方案**：

| 原因 | 检查方法 | 解决方案 |
|------|----------|----------|
| **AMD CPU** | `grep flags /proc/cpuinfo \| head -1 \| tr ' ' '\n' \| grep -E '^vmx$|^svm$'` 若输出 `svm` | AxVisor 仅支持 Intel VT-x，AMD 无法运行，需换 Intel 机器 |
| **WSL2 未启用嵌套虚拟化** | 检查 `~/.wslconfig` 是否有 `nestedVirtualization=true` | 添加配置后执行 `wsl --shutdown` 重启 |
| **KVM 未使用** | 确认使用 `--qemu-config .github/workflows/qemu-x86_64-kvm.toml` | 使用 KVM 配置，非 TCG |

### 7.6 KVM 权限问题

**现象**：`KVM: Permission denied`

**解决方案**：
```bash
# 添加用户到 kvm 组
sudo usermod -aG kvm $USER
# 重新登录或执行
newgrp kvm
```

### 7.7 NimbOS rootfs 注入失败

**现象**：`Failed to open .../qemu-x86_64 (NotFound)`

**解决方案**：
1. 安装 libguestfs-tools：`sudo apt install libguestfs-tools`
2. 或使用 sudo 权限挂载 loop 设备
3. 重新运行 `./scripts/setup_qemu.sh nimbos`

### 7.8 QEMU 无法启动

**现象**：`qemu-system-aarch64: command not found`

**解决方案**：
```bash
sudo apt-get install qemu-system-arm qemu-system-x86 qemu-utils
```
